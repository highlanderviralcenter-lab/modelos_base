# 1. VISÃO ARQUITETURAL EVOLUÍDA - VERSÃO COMPLETA (v2.0)

## 1.1 Princípios Fundamentais & Stack Tecnológico

### Princípios:

- **Consciência sobre Automação:** Núcleo é interpretação semântica, não execução cega.
- **Arquitetura Híbrida Humano-no-Loop:** Automação inteligente + execução manual controlada.
- **File-Centric Workflow:** Comunicação via pastas/arquivos estruturados.
- **Separação Radical:** Significado → Interpretação → Execução Manual → Síntese Automática.
- **Proteção de Originalidade:** Anti-Copyright Engine integrado em todas as saídas — cada vídeo é digitalmente único.
- **Resiliência por Design:** Retry automático, circuit breakers e health checks em todos os serviços.
- **Observabilidade Total:** Métricas de negócio e técnicas correlacionadas por project_id único.

### Stack Tecnológico Atualizado (v2.0):

| Camada | Tecnologia | Justificativa |
|--------|-----------|--------------|
| Backend Core | Python 3.12+, FastAPI, Pydantic v2 | Performance assíncrona, validação forte |
| Processamento de Vídeo | FFmpeg 6.0+ direto (sem MoviePy como compositor) | 3x mais rápido, 60% menos RAM |
| Composição de Alto Nível | MoviePy 2.x | Apenas para operações complexas de composição |
| Aceleração de Hardware | Intel VA-API (iHD driver) | Encode QSV sem GPU dedicada |
| Anti-Copyright Engine | cv2, librosa, scipy, soundfile | Fingerprint evasion imperceptível |
| Orquestração de Fluxo | Celery + Redis 7.x | Tasks assíncronas com persistência |
| File Watching | watchdog + inotify (Linux) | Eventos reais, sem polling |
| NLP / Arquétipos | transformers, KeyBERT, torch | Singleton com lazy loading |
| Armazenamento | PostgreSQL 15 + Redis 7 | Metadados + cache/filas |
| Monitoramento | Prometheus + Grafana | Métricas negócio + técnicas |
| Infraestrutura | Docker Compose (dev) → Kubernetes (prod) | Portabilidade total |

### 1.3.2 Anti-Copyright Engine — Especificação Técnica (NOVO)

**Propósito:** Garantir que cada vídeo gerado seja digitalmente único, evitando detecção por Content ID (YouTube), Rights Manager (Meta) e sistemas similares do TikTok. Todas as técnicas são imperceptíveis ao olho e ouvido humanos.

## 1.5 Fluxo de Dados Detalhado v2.0 (Sequência Completa)

```
sequenceDiagram
    participant User  as Usuário/Cliente
    participant NE    as Narrative Engine v2
    participant BG    as Batch Generator v2
    participant FS    as File System
    participant FW    as File Watcher v2

    User->>NE: Envia texto (transcrição/roteiro)
    NE->>NE: Carrega modelos via Singleton (1x por processo)
    NE->>NE: Analisa semântica + polaridade emocional
    NE->>NE: Mapeia para arquétipos (01-10) + confidence score
    NE->>NE: Gera ACE seed único para o projeto

    BG->>DB: Registra projeto (status: "aguardando_upload", schema_v: "2.0")
    BG->>User: Retorna caminho do projeto + instruções + schedule

    Note over User,FW: FASE MANUAL (Humano no Loop)
    User->>FS: Copia prompts para plataformas web gratuitas
    User->>FS: Faz download dos resultados
    User->>FS: Coloca arquivos em input_assets/video/ e /audio/

    FW->>FW: Valida nome, formato, tamanho, duração (ffprobe)
    FW->>FW: Verifica se lote está completo (10+10)

    alt Lote Completo
        FW->>RE: Dispara tarefa de renderização (via Redis)
```

## 1.9 Conclusão Técnica v2.0

Esta arquitetura representa a evolução do sistema ClipFusion com resolução de todos os problemas críticos identificados na v1:

**Problemas críticos resolvidos:**
- ✅ Anti-Copyright Engine com 13 técnicas (8 vídeo + 5 áudio) + metadata randomization + upload scheduling
- ✅ MoviePy substituído por FFmpeg direto como compositor (3x mais rápido, 60% menos RAM)
- ✅ Intel VA-API habilitado (i5-6200U + Intel HD 520 nativamente suportado)
- ✅ File Watcher com inotify real + wait_for_ready (fim do polling de 30s)
- ✅ NLP Singleton thread-safe (modelos carregados uma única vez)
- ✅ Health checks em todos os containers
- ✅ Redis com persistência e evição corretamente configurados
- ✅ Celery Workers separados com concurrency=2
- ✅ Correlation ID (project_id) em todo o stack de observabilidade
- ✅ CRF adaptativo por plataforma (era fixo em 23)
- ✅ Retry logic com backoff exponencial no File Watcher

**Capacidades novas v2.0:**
- Anti-Pattern Upload Scheduler — simula comportamento humano natural por plataforma
- Relatório ACE por projeto — rastreabilidade completa de transformações aplicadas
- Suporte a 5 plataformas de saída (era 3): TikTok, YouTube, Instagram Reels, Instagram Feed, Kwai
- schema_version no config.yaml — controle de versão de estrutura de projetos
- Métricas ACE no Prometheus — monitoramento de transformações aplicadas

O sistema está dimensionado para o hardware atual (i5-6200U, 8GB RAM, Intel HD 520, sem NVIDIA) e mantém caminho claro para escalar via containers sem re-arquitetura.
